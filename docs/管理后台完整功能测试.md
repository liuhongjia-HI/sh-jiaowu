## 自动化测试

一键执行：

```bash
./scripts/test-admin-functional.sh
```

该脚本会依次执行后端接口测试、前端 TypeScript 检查、Playwright 配置/用例静态检查和浏览器 E2E smoke。
如果本机有多个 Go 工具链，可用 `GO_BIN=/path/to/go ./scripts/test-admin-functional.sh` 指定。
CI 入口为 `.github/workflows/admin-functional-tests.yml`，可通过 Pull Request 或手动触发执行同一组后台功能测试。

### 前端浏览器 E2E

命令：

```bash
cd web
npm run test:e2e
```

说明：

- 默认 `npm run test:e2e` 使用 `playwright` core 的 smoke runner，测试已启动的管理端页面；执行前请先通过 `./start.sh` 或等价方式启动本地 API 和 Web。
- 首次运行如缺少浏览器，先执行 `cd web && node node_modules/playwright/cli.js install chromium`；如需使用本机 Chrome，可设置 `CHROME_EXECUTABLE_PATH=/Applications/Google\ Chrome.app/Contents/MacOS/Google\ Chrome`。
- 如需运行 `@playwright/test` spec，可执行 `npm run test:e2e:spec`，它会按 `web/playwright.config.ts` 自动启动本地 MySQL、`learning-api` 和 Vite 管理端。
- 如果已经通过 `./start.sh` 启动服务，`@playwright/test` 会复用现有服务。
- 如需指定 Go 工具链，可在执行前设置 `GO_BIN=/path/to/go`。
- 失败时会保留 trace 和截图，便于回放定位。

如需只做静态检查：

```bash
cd web
node node_modules/typescript/bin/tsc --noEmit --pretty false
node node_modules/typescript/bin/tsc --noEmit --pretty false --target ES2020 --module ESNext --moduleResolution Node --skipLibCheck --jsx react-jsx --types node playwright.config.ts tests/admin.e2e.spec.ts
ADMIN_SMOKE_DRY_RUN=1 node scripts/admin-smoke.mjs
```

覆盖范围：

| 模块 | 自动化场景 |
|---|---|
| 登录态 | 未登录访问后台跳转登录页；退出登录后无法继续访问后台 |
| 超级管理员 | 工作台、学生、套餐、开通、权限、课程、排课、资料、练习、批改、商业运营、通知、管理人员、教师、日志、设置页面均可打开 |
| 教师权限 | 教师菜单不展示商业运营、管理人员、系统设置；直达高权限 URL 显示无权限 |
| 校区管理员 | 学生管理、新增/导入入口、开通套餐、学习权限三类视图可用 |

## 人工验收清单

| 模块 | 验收步骤 | 期望结果 |
|---|---|---|
| 登录 | 用四类后台账号分别登录、刷新页面、退出登录 | 登录态保持正确；退出后回到登录页；角色名称显示正确 |
| 学生管理 | 新增学生、编辑手机号/年级/备注、停用账号、导入 CSV、点击提醒、查看详情 | 列表刷新；错误 CSV 显示失败条数；提醒有成功反馈；详情展示开通、权限、学习记录 |
| 学习套餐 | 新建套餐，绑定学习空间和内容类型，再编辑状态/摘要 | 套餐列表展示正确；编辑后开通预览同步变化 |
| 开通套餐 | 选择学生和套餐，查看权限预览，确认开通，再重复选择同套餐 | 首次提示开通成功；重复开通按钮置为已开通并显示有效期 |
| 学习权限 | 切换按学生、按套餐、按内容视图，搜索学生/套餐/内容 | 搜索结果准确；课程、资料、练习开放范围一致 |
| 课程内容 | 新建课程，选择学习空间，编辑章节数和状态 | 年级/学科由学习空间派生；列表更新 |
| 学习资料 | 上传 PDF/Word/PPT，编辑标题/课程/章节/状态，点击预览和下载 | 上传成功；草稿/停用状态正确；PDF 可预览；下载能拿到原文件 |
| 课后练习 | 上传练习文件，编辑截止日期和状态 | 列表展示截止日期；停用后学生端不可见 |
| 批改反馈 | 打开待批改记录，填写分数、评语、奖励并提交 | 成功后记录从待批改列表移除；学生端可见结果 |
| 通知提醒 | 教师发送负责学科通知，管理员发送全部学生通知 | 后台列表出现新通知；学生端按范围过滤 |
| 排课管理 | 维护老师/学生可上课时间，生成候选，确认成班，调课，取消 | 候选显示可排和缺席学生；课表更新；取消后状态正确 |
| 商业运营 | 创建订单，登记收款、退款、合同、发票、课消、续费提醒、家长通知 | 统计卡刷新；全额收款后学习权限自动开通；超额课消被拒绝 |
| 教师管理 | 新增教师，分配课程范围，切换上传和批改权限，停用账号 | 教师账号权限立即影响内容维护能力 |
| 管理人员 | 超级管理员新增运营/校区管理员，编辑岗位和校区，停用账号 | 非超级管理员不能访问；至少保留一个正常超级管理员 |
| 系统设置 | 修改学年、年级、学期、水印和下载规则 | 设置保存成功；操作日志记录操作者和变更 |
| 操作记录 | 完成新增、编辑、重置密码、登录失败等操作后查看日志 | 日志包含操作人、动作、对象、时间和关键详情 |

## 上线前补充验证

- 在微信开发者工具或真机中验证资料预览、文件下载和学生端可见性。
- 在生产等价环境验证 `CORS_ALLOWED_ORIGINS`、`AUTH_TOKEN_SECRET`、`MYSQL_DSN`、微信凭据和演示登录开关。
- 若接入真实支付、合同、发票系统，需用沙箱账号补充外部系统回调和失败重试测试。
