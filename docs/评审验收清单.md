## 管理端核心链路

| 链路 | 当前状态 | 验收证据 |
|---|---|---|
| 学生管理 | 支持新增、编辑、导入、提醒、查看详情与学习记录 | `GET/POST/PUT /api/students`、`POST /api/students/import`、`POST /api/students/{id}/remind` |
| 学习套餐 | 支持创建、编辑、绑定学习空间和开放内容类型 | `POST /api/packages`、`PUT /api/packages/{id}` |
| 开通套餐 | 支持开通前预览、重复开通识别、权限同步 | `GET /api/grants/preview` 返回 `alreadyOpened` / `existingUntil`，`POST /api/grants` |
| 学习权限 | 支持按学生、套餐、内容核对开放范围 | `GET /api/permissions/students`、`/packages`、`/content` |
| 课程内容 | 支持课程创建、编辑和范围校验 | `POST /api/courses`、`PUT /api/courses/{id}` |
| 学习资料 | 支持上传、编辑、草稿/启用/停用、预览、下载 | `POST /api/materials`、`PUT /api/materials/{id}`、`GET /api/files/{id}/preview` |
| 课后练习 | 支持上传、编辑、草稿/启用/停用、学生任务可见性 | `POST /api/homework`、`PUT /api/homework/{id}` |
| 批改反馈 | 支持教师填写分数、评语、奖励，并生成学生可见结果 | `POST /api/reviews/{id}/complete` |
| 通知提醒 | 支持后台发送通知，学生端按范围过滤 | `POST /api/notices`、`GET /api/student/notices` |
| 排课管理 | 支持师生可用时间、候选排课、确认成班、调课、取消 | `GET/PUT /api/availability`、`POST /api/scheduling/candidates`、`POST/PUT /api/schedule-classes` |
| 系统设置 | 支持校区管理员/超级管理员编辑设置并记录日志 | `PUT /api/settings` |

## 学生端核心链路

| 链路 | 当前状态 | 验收证据 |
|---|---|---|
| 登录与登录态 | 支持演示账号登录；401 自动清理登录态并跳转登录页 | `POST /api/auth/demo-student-login`，`miniprogram/utils/request.js` |
| 学习首页/学习页 | 展示已开通课程、资料和真实进度 | `GET /api/student/home`、`GET /api/student/study` |
| 课程详情 | 展示课程、资料、练习和学习地图 | `GET /api/student/study/{id}` |
| 资料预览 | 支持资料详情和预览地址 | `GET /api/student/materials/{id}` |
| 小挑战任务 | 展示待完成/已完成状态 | `GET /api/student/tasks` |
| 答题草稿 | 本地保存草稿，提交成功后清理 | `miniprogram/pages/answer/index.js` |
| 提交与结果 | 提交后自动判分，结果页展示分数、评语、奖励 | `POST /api/student/submissions`、`GET /api/student/submissions/{id}` |
| 成长与徽章 | 按学习数据派生成长记录和徽章 | `GET /api/student/growth`、`GET /api/student/badges` |
| 收藏 | 支持资料和小挑战收藏/取消收藏 | `GET/POST/DELETE /api/student/favorites` |
| 可上课时间 | 学生可维护时间并查看课表 | `GET/PUT /api/student/availability`、`GET /api/student/schedule` |

## 评审结论

当前功能已覆盖教务后台、教师内容维护、学生学习闭环、权限同步、通知、排课和系统设置等主要业务路径。按功能完整度评估，已达到约 95% 的可评审状态。

剩余风险：

- 真机文件预览仍需在微信开发者工具或真机环境验证 PDF / Word / PPT 打开体验。
- “联系学习伙伴”仍属于产品形态未定功能，当前不作为核心评审链路入口。
